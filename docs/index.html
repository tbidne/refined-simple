refined-simple-0.1.0.0: A library that provides a simple implementation of refined types.

refined-simple provides a simple implementation of refined types with a small dependency footprint. Its main inspirations are the wonderful refined and facts packages.

Differences from refined:

  • Only dependencies are TemplateHaskell and Text, giving refined-simple a significantly lighter dependency footprint.
  • Predicate composition is a type-level list (like facts), not a combination of logical operators. This makes composition less general than refined's, but it is easier to work with.
  • Polymorphism over predicates.

Differences from facts:

  • Fewer dependencies (though facts is also light).
  • Simpler API.
  • More built-in functionality (e.g. predicates for foldables, Numbers, Text).

In short: If you want a "batteries included" refined library with the highest ecosystem integration and most flexible composition (and don't mind some transitive dependencies you may not use e.g. aeson), check out refined. If instead you want a more compact design with a strong mathematical foundation, try facts. Finally, if you would ordinarily use smart constructors but are tired of some of the shortcomings (e.g. high boilerplate, no safe creation for compile-time constants, no composition), then perhaps give refined-simple a try.
